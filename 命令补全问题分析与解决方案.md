# 命令补全问题分析与解决方案

## 一、问题描述

在 astrbot 等聊天机器人框架中，机器人连接 QQ/Telegram/Discord 后，用户在聊天输入框输入 `/` 时会自动弹出可用命令列表。而 sillytavern-gateway 的自定义插件注册的命令（包括内置的 `/help`、`/status`、`/clear`）均无法在输入 `/` 时显示补全列表。

## 二、根因分析

### 2.1 两套互不相通的命令系统

项目中存在两套完全独立的斜杠命令系统：

| 系统 | 注册方式 | 触发位置 | 补全支持 |
|------|----------|----------|----------|
| ST 扩展斜杠命令 | `SlashCommandParser.addCommandObject()` | SillyTavern 网页输入框 | ST UI 原生支持 |
| 网关插件命令 | `static commands` + `CommandRouter.register()` | QQ/TG/Discord 聊天消息 | **无任何补全机制** |

ST 扩展端（`index.js` 第 820-881 行）只注册了 `/gateway` 一个命令到 SillyTavern 的 `SlashCommandParser`。网关插件命令系统（`CommandRouter`）仅做"收到 `/xxx` 消息后匹配并执行"，不涉及任何补全推送。

### 2.2 各适配器缺少平台原生命令注册调用

对三个平台适配器逐一审查：

- **Telegram 适配器**（`telegram-adapter.js`）：使用 `node-telegram-bot-api`，但**从未调用 `bot.setMyCommands()`**。这是 Telegram Bot API 中让 `/` 触发命令提示的唯一方法。
- **Discord 适配器**（`discord-adapter.js`）：使用 `discord.js`，但**从未调用 `client.application.commands.set()`** 注册 Application Commands。
- **QQ OneBot 适配器**（`onebot-adapter.js`）：纯 WebSocket 消息收发，OneBot v11 协议本身**没有标准化的命令注册 API**。

### 2.3 命令元数据仅用于 `/help` 文本

`CommandRouter.listCommands()` 方法（`command-router.js` 第 130-142 行）虽然能返回所有命令列表，但它**只被内置 `/help` 命令内部使用**，没有被任何 API 端点暴露，也没有被推送到任何平台。

全局搜索 `setMyCommands|registerCommands|applicationCommands|syncCommands` 在整个项目中除了 `CommandRouter` 内部的 `this.commands.set()` Map 操作外，没有任何平台命令注册调用。

### 2.4 结论

**这不是"自定义插件命令"特有的问题，而是全项目级的缺失功能。** 包括内置的 `/help`、`/status`、`/clear` 在内的所有网关命令都不会出现在任何平台的 `/` 补全列表中。

如果用户在 QQ 或 Telegram 中看到了 `/` 补全，那是因为：
- QQ 端：NapCat/Lagrange 等客户端自身有命令补全配置（在客户端侧手动配置）
- Telegram 端：可能通过 @BotFather 手动 `setcommands` 配置过
- 这些都是**平台侧手动配置**，与网关代码无关

## 三、解决方案

### 3.1 总体思路

在网关中新增一个「命令同步」机制：
1. 适配器连接成功后，自动将 `CommandRouter` 中所有命令推送到平台的原生命令注册 API
2. 插件启用/禁用/重载后，自动重新同步
3. 提供 REST API 端点供手动触发同步

#### 3.3.1 base-adapter.js — 新增 syncCommands 虚方法

```js
 * 使得用户在聊天输入 "/" 时能看到可用命令列表。

 * @returns {Promise<boolean>} 是否同步成功

```

#### 3.3.2 telegram-adapter.js — 调用 setMyCommands

在 `TelegramAdapter` 类中添加 `syncCommands` 方法：

```js
/**
 * 同步命令列表到 Telegram
 * 调用 Bot API setMyCommands，使用户在聊天框输入 "/" 时弹出命令列表
 *
 * @param {Array<{name: string, description: string}>} commands - 网关命令列表
 * @returns {Promise<boolean>}
 */
async syncCommands(commands) {
    if (!this.bot) {
        this.logger.warn('Telegram Bot 未初始化，无法同步命令');
        return false;
    }

    try {
        // Telegram 命令名限制：小写字母+数字+下划线，最长 32 字符
        const tgCommands = [];
        const seen = new Set();

        for (const cmd of commands) {
            // 转换命令名：只保留 a-z 0-9 _，其余替换为下划线
            let name = cmd.name.toLowerCase().replace(/[^a-z0-9_]/g, '_');
            name = name.replace(/^_+/, '').substring(0, 32);

            if (!name || seen.has(name)) continue;
            seen.add(name);

            // 描述截断到 256 字符
            const desc = (cmd.description || cmd.name).substring(0, 256);
            tgCommands.push({ command: name, description: desc });
        }

        // Telegram 限制最多 100 个命令
        const trimmed = tgCommands.slice(0, 100);

        // 私聊补全
        await this.bot.setMyCommands(trimmed, { scope: { type: 'all_private_chats' } });
        this.logger.info(`已同步 ${trimmed.length} 个命令到 Telegram（私聊补全）`);

        // 群聊补全（部分客户端支持）
        try {
            await this.bot.setMyCommands(trimmed, { scope: { type: 'all_group_chats' } });
            this.logger.debug('已同步命令到 Telegram 群聊补全');
        } catch (_) {
            // 群聊 scope 可能不支持，忽略
        }

        return true;
    } catch (error) {
        this.logger.error(`同步命令到 Telegram 失败: ${error.message}`);
        return false;
    }
}
```

**Telegram 命令名转换规则**：
- 中文名（如 `帮助`）会被替换为下划线再去除前导下划线，导致变为空名被跳过
- 因此中文别名的命令需要在 `CommandRouter` 层面额外提供英文/拼音名
- 建议在命令定义中同时声明英文 `name` 和中文 `alias`

#### 3.3.3 discord-adapter.js — 注册 Application Commands

在 `DiscordAdapter` 类中添加 `syncCommands` 方法：

```js
import { SlashCommandBuilder, REST, Routes } from 'discord.js';

/**
 * 同步命令列表到 Discord
 * 注册全局 Application Commands，使 "/" 输入时弹出命令列表
 *
 * @param {Array<{name: string, description: string}>} commands - 网关命令列表
 * @returns {Promise<boolean>}
 */
async syncCommands(commands) {
    if (!this.client?.application?.id) {
        this.logger.warn('Discord 客户端未就绪，无法同步命令');
        return false;
    }

    try {
        // Discord 命令名限制：小写字母+数字+下划线+连字符，1-32 字符
        const slashCommands = [];
        const seen = new Set();

        for (const cmd of commands) {
            let name = cmd.name.toLowerCase().replace(/[^a-z0-9_-]/g, '_');
            name = name.replace(/^[_-]+/, '').substring(0, 32);

            if (!name || name.length < 1 || seen.has(name)) continue;
            seen.add(name);

            const desc = (cmd.description || cmd.name).substring(0, 100);

            slashCommands.push(
                new SlashCommandBuilder()
                    .setName(name)
                    .setDescription(desc)
                    .toJSON()
            );
        }

        // 使用 REST API 注册全局命令（所有服务器生效，有 1 小时缓存延迟）
        const rest = new REST({ version: '10' }).setToken(this.config.botToken);
        await rest.put(
            Routes.applicationCommands(this.client.application.id),
            { body: slashCommands }
        );

        this.logger.info(`已同步 ${slashCommands.length} 个命令到 Discord`);
        return true;
    } catch (error) {
        this.logger.error(`同步命令到 Discord 失败: ${error.message}`);
        return false;
    }
}
```

**Discord 注意事项**：
- 全局命令有最多 1 小时的传播延迟
- 如果需要即时生效，可注册 guild commands（指定服务器 ID）
- Discord 的 Application Commands 交互模型与网关的文本命令不同，用户点击补全命令后 Discord 会发送一个 `interaction` 事件而非普通消息。需要在适配器中额外处理 `Events.InteractionCreate` 事件，将 interaction 转换为普通文本消息再走 `CommandRouter` 路由

#### 3.3.4 onebot-adapter.js — QQ 平台说明

OneBot v11 协议**没有标准化的命令注册 API**。QQ 的命令补全体验取决于具体的 OneBot 实现：

| OneBot 实现 | 命令补全方式 |
|---|---|
| NapCat | 支持通过配置文件或扩展 API 注册命令提示，但非标准 |
| Lagrange | 类似 NapCat，有扩展但不统一 |
| go-cqhttp | 已停止维护，无此功能 |

**建议方案**：在 `OneBotAdapter` 中添加 `syncCommands` 方法，尝试调用 NapCat 的扩展 API，失败则记录日志：

```js
/**
 * 同步命令列表到 QQ（OneBot）
 * OneBot v11 无标准化命令注册 API，此处尝试 NapCat 扩展，失败则跳过
 */
async syncCommands(commands) {
    // OneBot v11 标准中没有命令注册 API
    // NapCat 等实现可能有扩展，但接口不统一
    // 此处仅记录日志，实际补全需在客户端侧手动配置
    this.logger.info(`QQ 平台暂不支持自动命令同步，共 ${commands.length} 个命令。`);
    this.logger.info('如需命令补全，请在 OneBot 客户端（NapCat/Lagrange）侧手动配置。');
    return false;
}
```

#### 3.3.5 command-router.js — 新增 getCommandsForSync()

在 `CommandRouter` 类中添加辅助方法：

```js
/**
 * 获取所有命令（适配器同步用格式）
 * @returns {Array<{name: string, description: string, usage: string, plugin: string}>}
 */
getCommandsForSync() {
    const list = [];
    for (const [name, cmd] of this.commands) {
        list.push({
            name: name,
            description: cmd.description || '',
            usage: cmd.usage || '',
            plugin: cmd.pluginName,
        });
    }
    return list;
}
```

#### 3.3.6 gateway-core.js — 新增 syncAllCommands + 连接事件钩子

在 `GatewayCore` 类中添加：

```js
/**
 * 同步命令列表到所有已连接平台
 * @param {Array<{name: string, description: string}>} commands - 命令列表
 */
async syncAllCommands(commands) {
    for (const [name, adapter] of this.adapters) {
        if (adapter.isConnected()) {
            try {
                await adapter.syncCommands(commands);
            } catch (error) {
                logger.error(`同步命令到 ${name} 失败: ${error.message}`);
            }
        }
    }
}
```

在 `registerAdapter()` 方法中，`adapter.on('connected', ...)` 回调内追加：

```js
adapter.on('connected', () => {
    logger.info(`[${name}] 已连接`);
    this.emit('adapterConnected', name);

    // 连接成功后自动同步命令列表
    // 需要外部注入 commandRouter 引用
    if (this._commandRouter) {
        const commands = this._commandRouter.getCommandsForSync();
        adapter.syncCommands(commands).catch(err => {
            logger.error(`[${name}] 命令同步失败: ${err.message}`);
        });
    }
});
```

同时添加 setter 方法供 `PluginManager` 注入 `CommandRouter`：

```js
/**
 * 注入命令路由器引用（供命令同步使用）
 */
setCommandRouter(router) {
    this._commandRouter = router;
}
```

#### 3.3.7 plugin-manager.js — 插件变更后触发重新同步

在 `PluginManager` 构造函数中保存 `gatewayCore` 引用（已有）。

在 `init()` 方法末尾注入 `CommandRouter`：

```js
async init() {
    // ... 现有代码 ...

    // 注入 CommandRouter 到 GatewayCore，用于连接时自动同步命令
    this.gateway.setCommandRouter(this.commandRouter);

    // 初始同步：对已连接的适配器立即同步一次
    const commands = this.commandRouter.getCommandsForSync();
    await this.gateway.syncAllCommands(commands);

    logger.info(`插件系统初始化完成，已加载 ${this.loader.getAllPlugins().size} 个插件`);
}
```

在 `enablePlugin`、`disablePlugin`、`reloadPlugin`、`installFromGitHub` 方法中，注册/注销命令后追加重新同步：

```js
// 例如 enablePlugin 末尾：
async enablePlugin(name) {
    // ... 现有代码 ...
    this.registerPlugin(instance);

    // 重新同步命令到所有平台
    const commands = this.commandRouter.getCommandsForSync();
    await this.gateway.syncAllCommands(commands);

    logger.info(`插件已启用: ${name}`);
    return { success: true, message: `插件 ${name} 已启用` };
}
```

对 `disablePlugin`、`reloadPlugin`、`installFromGitHub` 做同样处理：在命令注册状态变更后调用 `this.gateway.syncAllCommands(commands)`。

#### 3.3.8 server/index.js — 手动同步 API 端点

在 API 路由区域添加：

```js
/**
 * 手动同步命令列表到所有平台
 */
app.post('/api/gateway/sync-commands', async (req, res) => {
    if (!pluginManager) {
        return res.status(500).json({ success: false, error: '插件系统未初始化' });
    }
    const commands = pluginManager.commandRouter.getCommandsForSync();
    await gatewayCore.syncAllCommands(commands);
    res.json({ success: true, message: `已同步 ${commands.length} 个命令到所有已连接平台` });
});
```

### 3.4 命令同步时机

| 时机 | 触发方式 | 说明 |
|------|----------|------|
| 网关启动后 | `PluginManager.init()` 末尾 | 初始同步，覆盖已连接的适配器 |
| 适配器重连成功 | `adapter.on('connected')` 事件 | 适配器断线重连后自动补同步 |
| 插件启用 | `enablePlugin()` | 新命令需要注册到平台 |
| 插件禁用 | `disablePlugin()` | 移除已禁用插件的命令 |
| 插件重载 | `reloadPlugin()` | 命令可能变更 |
| GitHub 安装 | `installFromGitHub()` | 新插件安装后 |
| 手动触发 | `POST /api/gateway/sync-commands` | 调试或修复用 |

### 3.5 中文命令名处理

网关插件命令大量使用中文名和中文别名（如 `/帮助`、`/抢答`、`/状态`）。各平台对命令名的限制：

| 平台 | 命令名字符限制 | 中文支持 |
|------|---------------|---------|
| Telegram | `[a-z0-9_]`，最长 32 字符 | 不支持 |
| Discord | `[-_a-zA-Z0-9]`，1-32 字符 | 不支持中文 |
| QQ | 取决于客户端实现 | 部分支持 |

**建议方案**：在命令定义中，确保每个命令都有英文 `name`，中文放在 `alias` 中。例如：

```js
static commands = [
    {
        name: 'bid',           // 英文主名，用于平台补全注册
        alias: ['抢答'],        // 中文别名，仅网关内部路由用
        description: '抢答剧情推进',
        handler: 'handleBid',
    },
    {
        name: 'skip',
        alias: ['跳过'],
        description: '跳过当前抢答轮次',
        handler: 'handleSkip',
    },
    {
        name: 'status',
        alias: ['状态'],
        description: '查看当前抢答状态',
        handler: 'handleStatus',
    },
];
```

如果命令确实只有中文名（如内置的 `/帮助`），在 `syncCommands` 中会被跳过。需要为内置命令也补充英文名，或修改 `registerBuiltinCommands` 让 `name` 使用英文、`alias` 使用中文。

### 3.6 Discord 交互事件处理补充

Discord 的 Application Commands 与文本命令的交互模型不同。用户点击补全命令后，Discord 发送的是 `InteractionCreate` 事件而非普通消息。需要在 `discord-adapter.js` 的 `setupEventHandlers()` 中添加：

```js
// 处理斜杠命令交互
this.client.on(Events.InteractionCreate, async (interaction) => {
    if (!interaction.isChatInputCommand()) return;

    // 将 interaction 转换为标准入站消息
    const args = interaction.options.data.map(opt => opt.value).join(' ');
    const content = `/${interaction.commandName}${args ? ' ' + args : ''}`;

    const inboundMsg = new InboundMessage({
        platform: 'discord',
        messageId: interaction.id,
        chatId: interaction.channelId,
        chatType: interaction.guildId ? 'group' : 'private',
        senderId: interaction.user.id,
        senderName: interaction.user.username,
        content: content,
        timestamp: Date.now(),
        raw: interaction,
    });

    // 标记为交互类型，供后续 reply 使用
    inboundMsg._interaction = interaction;

    this.emit('message', inboundMsg);

    // 回复 interaction（避免 Discord 超时报错）
    await interaction.deferReply();
});
```

然后在 `PluginContext` 或适配器的 `send()` 方法中，检查 `message._interaction` 是否存在，如果存在则用 `interaction.editReply()` 回复而非 `channel.send()`。
